Using the DNS relay


Manage the host table

Tftpd32 uses the API getaddrinfo to resolve the domain name.
This API tries to find the host name in the file
%SystemRoot%\System32\drivers\etc\hosts
(or the file pointed by the registry key
\HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Tcpip\Parameters\DataBasePath).

Populate this file with your hosts and Tftpd32 will propagate the info.
If getaddrinfo founds no suitable host, it relays the request to the DNS server.
